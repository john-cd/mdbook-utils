## 2024-05-18 - Fix Path Traversal in Code Extraction
**Vulnerability:** In `src/markdown/extract_code.rs`, code blocks extracted from Markdown files were written to dynamically constructed filenames like `code_dest_dir_path.join(code_filename)`. Even though `code_filename` derived from `file_stem` currently didn't allow directory traversal directly through user inputs, missing path containment validation is a known high-severity class in this codebase (defense-in-depth).
**Learning:** Canonicalization of dynamically constructed paths (and validating they remain within the intended base directory using `crate::fs::is_path_within`) is critical. A specific quirk encountered is that `fs::canonicalize` on a non-existent target file fails. Thus, the file must be eagerly created *before* path canonicalization to properly perform the validation.
**Prevention:** Always validate constructed output paths using canonicalization constraints (e.g. `is_path_within`) against the permitted directory. When validating new output files, either check the parent directory or create the file first before canonicalizing.
